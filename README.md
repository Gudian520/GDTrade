# GD Trade

GD Trade 是一个面向 A 股持仓监控与决策辅助的 Android 原生 App。当前 V1 只做持仓管理、观察池、风险提示、交易记录和人工确认入口，不实现自动证券交易。

## V1 边界

- 不自动提交真实证券买卖订单。
- 不存储证券账户密码、短信验证码、Cookie 或任何交易登录凭据。
- 行情数据用于研究辅助，界面必须标注数据来源；接口失败时回退本地静态样例，不能声称样例数据为实时行情。
- 风险引擎可以否决买入信号，UI 只展示提示和人工确认入口。
- “打开同花顺”只用于跳转人工确认，不代表下单。

## 当前功能

- 首页策略驾驶舱。
- 当前持仓：
  - 华天科技 002185：200 股
  - 华夏中证人工智能主题 ETF 515070：800 份
  - 京东方 A 000725：100 股
- 动态观察池与信号状态展示。
- 交易记录：
  - 2026-07-13 华天科技 24.62 元卖出 100 股
  - 2026-07-13 京东方 A 6.92 元卖出 200 股
- 一键打开同花顺进行人工确认。
- 本地数据维护：新增/删除持仓、观察股票和交易记录，支持恢复样例数据。
- 行情接口：尝试读取腾讯行情接口，显示价格、涨跌幅和接口时间；失败时回退本地静态样例。
- ChatGPT Plus 辅助：生成可复制的分析提示词，粘贴到 ChatGPT Plus 使用；不要求 OpenAI API Key。

## ChatGPT Plus 使用方式

当前默认采用“方案 3”：App 只生成分析提示词，不直接调用 OpenAI API。

1. 在首页点击“刷新行情”，更新当前参考行情。
2. 点击“复制给ChatGPT”。
3. 打开 ChatGPT Plus，把剪贴板里的提示词粘贴进去。
4. 让 ChatGPT 基于持仓、观察池、交易记录和风险条件输出研究辅助意见。
5. 所有结论都必须再回到行情软件和人工判断中确认，不能自动下单。

这种方式不需要 API Key，也不会把 API Key 存在手机里。

## 本地数据维护

App 会使用 Room Database 把用户维护的数据保存在手机本地应用私有存储中：

- 新增持仓会覆盖同代码的旧持仓。
- 新增观察股票会覆盖同代码的旧观察项。
- 新增交易记录会追加到记录列表顶部。
- 删除按钮只删除当前条目。
- “恢复样例”会把持仓、观察池和交易记录恢复为 V1 样例数据。
- 从旧版本升级时，App 会一次性把原 SharedPreferences 数据迁移到 Room；迁移成功后旧数据保留作回退，但不再参与日常读写。卸载 App 会清除本地保存的数据。

当前 V1 仍不连接证券账户，也不保存任何证券登录凭据。

## 行情接口

V1 会尝试通过腾讯行情接口读取持仓股票的参考价格、涨跌幅和接口时间。该数据可能存在延迟或不可用，App 会在界面中标注来源；接口失败时回退本地静态样例数据。

行情数据仅用于持仓监控与人工决策辅助，不构成交易建议，也不会触发自动下单。

## 后端代理（可选）

仓库中保留了可选的 `backend/gpt_proxy.py` 示例，用于未来在服务器端安全调用 OpenAI API。当前 App 默认不依赖它。

如未来启用后端代理，手机端仍不应保存 OpenAI API Key；API Key 只能放在后端环境变量中。

## 架构

项目采用分层结构：

- `domain/model`：持仓、候选股票、行情、交易记录等领域模型。
- `domain/risk`：风险引擎与风险决策。
- `data/repository`：仓储接口与 V1 本地实现。
- `data/ai`：可选 GPT 后端代理接口代码。
- `ui/dashboard`：策略驾驶舱界面、UI 状态与 ViewModel。
- `ui/theme`：Compose 主题。

UI 状态由 `DashboardViewModel` 使用 `StateFlow<DashboardUiState>` 管理。仓储层通过接口抽象，后续可替换为真实行情或本地数据库实现。

## 信号状态

V1 支持以下信号状态：

- 持有观察
- 减仓风险提示
- 暂不追高
- 等待回调
- 出现研究型买点
- 高位不追
- 仅观察

## 本地运行

推荐使用 Android Studio 打开项目并同步 Gradle，然后连接 Android 手机运行。

命令行构建：

```powershell
.\gradlew test
.\gradlew assembleDebug
```

远程 GitHub Actions 会使用 Java 17 和 Gradle 8.10.2 运行：

```powershell
gradle test --no-daemon
gradle assembleDebug --no-daemon
```
